# Fix: Lombok Setters Not Recognized by IDE

## Problem
You're getting: `java: cannot find symbol method setSaleId(java.lang.Long)`

This means your IDE (IntelliJ IDEA) is not recognizing the setters generated by Lombok's `@Data` annotation.

## Root Cause
IntelliJ needs to have **Lombok annotation processing enabled** to recognize Lombok-generated methods at compile time.

## Solution

### Method 1: Enable Lombok Annotation Processing (RECOMMENDED)

1. **Open IntelliJ IDEA Settings:**
   - File → Settings (on Windows/Linux)
   - IntelliJ IDEA → Preferences (on macOS)

2. **Navigate to Annotation Processing:**
   - Search for "Annotation Processors" in the search box
   - Or go to: Build, Execution, Deployment → Compiler → Annotation Processors

3. **Enable the following checkboxes:**
   ✅ Enable annotation processing
   ✅ Obtain processors from project classpath

4. **Click Apply and OK**

5. **Rebuild the project:**
   - Build → Rebuild Project
   - Or press Ctrl+F9

### Method 2: Install Lombok Plugin (Alternative)

If the annotation processing doesn't work:

1. **Open IntelliJ Plugins:**
   - File → Settings → Plugins (on Windows/Linux)
   - IntelliJ IDEA → Preferences → Plugins (on macOS)

2. **Search for "Lombok"**

3. **Install the "Lombok" plugin** by JetBrains

4. **Restart IntelliJ IDEA**

### Method 3: Update pom.xml (If needed)

Make sure your `pom.xml` has the correct Lombok dependency. Check that it includes:

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.30</version>
    <scope>provided</scope>
</dependency>
```

And ensure the Maven compiler plugin has annotation processing enabled:

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-compiler-plugin</artifactId>
    <version>3.11.0</version>
    <configuration>
        <source>17</source>
        <target>17</target>
        <annotationProcessorPaths>
            <path>
                <groupId>org.projectlombok</groupId>
                <artifactId>lombok</artifactId>
                <version>1.18.30</version>
            </path>
        </annotationProcessorPaths>
    </configuration>
</plugin>
```

## What This Does

Once enabled, IntelliJ will:
- ✅ Recognize `@Data` annotation
- ✅ Show getters and setters in autocomplete
- ✅ Allow calling `setSaleId()`, `setVendorId()`, etc.
- ✅ Highlight code correctly without red underlines

## Why This Happens

Lombok uses **annotation processing** to generate code at compile time. Without enabling this in the IDE:
- The compiler still generates the code (build works fine)
- But the IDE doesn't know about the generated methods
- So it shows false errors (red squiggles)

The actual compilation and execution work fine because Maven's compiler plugin has annotation processing enabled in `pom.xml`.

## TL;DR

**Quick Fix:**
1. File → Settings → Build, Execution, Deployment → Compiler → Annotation Processors
2. ✅ Check "Enable annotation processing"
3. Build → Rebuild Project
4. Done!

Your code is correct - just need to tell IntelliJ to recognize Lombok's generated code.
